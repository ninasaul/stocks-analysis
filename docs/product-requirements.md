# 智谱投研

**文档版本**：1.0（2026-04-16）  
**文档性质**：产品与设计基线全文；接口与数据字段的落地格式以实现侧契约为准，正文定义能力范围与验收口径；修订见文末版本表。

本文档定义「智谱投研」的产品设计基线（仓库 stocks-analysis-simple），聚焦“单只股票分析”场景下的择时研究输出。目标是将复杂分析过程转化为可执行的观察与风控方案，优先解决“什么时候关注、什么时候等待、什么时候规避风险”。

---

## 1. 产品定位

**产品定义**  
智谱投研面向个人投资者与研究型交易者的单票择时分析助手。输入一只股票与分析周期，输出结构化择时结论和风控计划。

**核心价值**  
从“给信息”升级到“给决策建议”，并明确建议的触发条件、失效条件和风险边界。

**一句话定位**  
用统一评分与多维信号融合，帮助用户在单只股票上做出更稳健的择时判断与风险识别。

**流程关系**  
用户可先通过「问答对话式选股」与系统多轮交流，收敛到少数候选或一只股票，再进入单票择时分析；选股与择时均为研究辅助，不涉及交易执行。

---

## 2. 目标用户与场景

### 2.1 目标用户

- **短中线交易者**：关注 1 天到 3 个月的交易窗口，需要明确时点建议。
- **进阶投资者**：理解技术面与基本面，但希望减少主观判断偏差。
- **内容型投研用户**：需要快速产出结构化观点，用于复盘和沟通。

### 2.2 高频使用场景

- 盘后评估“明日是否可试仓”。
- 持仓过程中评估“是否加仓或减仓”。
- 突发事件后评估“原计划是否失效”。
- 关键财报窗口评估“是否降低仓位等待不确定性落地”。
- 不确定具体标的时，通过对话向系统说明偏好与约束，由系统追问澄清后给出可继续深入分析的候选标的。

## 3. 产品目标与非目标

### 3.1 当前阶段目标（MVP）

1. 支持单只股票输入并输出标准化择时报告。
2. 提供五态决策结果：观望、试仓、加仓、减仓、离场。
3. 给出可执行研究计划：关注区间、风险位、观察目标位、失效条件、风险敞口建议。
4. 输出置信度与风险等级，帮助用户控制决策预期。
5. 支持建议记录与复盘，形成可回看、可检验闭环。
6. 提供问答对话式选股模块：以对话流收集并澄清偏好与约束，输出候选标的及简要依据，并可一键进入单票择时报告。
7. 提供账号体系：**手机号 + 密码**、**手机号 + 短信验证码**、**微信扫码**三种方式完成注册与登录，并支持方式间绑定到同一账号。
8. 提供**用户订阅机制**：可购买的订阅档位、权益与用量边界、开通/续费与到期处理；与核心分析能力打通（见 **FR-012～FR-014**）。
9. 提供 **Landing 首页**及可公开访问的**隐私政策**、**用户服务条款**等静态页，作为品牌入口与合规基线（见 **FR-015～FR-017**、**7.11**）。

### 3.2 非目标（当前阶段不做）

- 多票组合优化与资产配置建议。
- 自动下单、委托管理、成交回报与券商交易执行。
- 个性化投顾与收益承诺。
- 复杂衍生品策略（期权、两融、套利）。
- 证券经纪开户、代客理财、以投资收益分成或「保证收益」为对价的收费模式。
- 首期**不以邮箱或海外手机号为主账号**（与 **FR-009** 大陆 11 位一致）；若后续扩展，须另附数据与合规评估。

## 4. 核心设计原则

1. **可执行优先**：结论必须能转换成“价格区间 + 风险敞口 + 风险位”。
2. **可解释优先**：每条建议都应能追溯到信号来源与评分依据。
3. **风险优先于收益**：未满足风控前提时，默认给出保守建议。
4. **一致性优先**：同一输入在同一数据快照下，输出应稳定可复现。
5. **简洁优先**：避免冗长叙述，优先结构化字段与清晰状态机。

## 5. 关键能力设计

### 5.0 选股模块（问答对话形态）

选股不采用复杂筛选面板，而以**基于大模型的 AI 连续对话**完成意图澄清与约束收集（前端交互见 **7.6**；**Agent 编排逻辑**见 **5.6**）：**既可由系统主动追问，也可由用户先用自然语言提问或描述需求**；系统结合上下文理解意图后，以自然语言作答，并在同一条或紧随其后的助手消息中提供**可点选的选项**（如按钮、chips、单选/多选列表），引导用户收敛到明确约束。用户也可仅用点选或短文本补充。底层仍将对话内容收敛为 **D1～D9 结构化偏好快照**，保证可解释性与可复现。

#### 交互形态

- 对话以时间线展示（用户消息与助手消息交替），符合常见即时对话习惯。
- 系统可主动追问缺失信息、复述已理解约束并请用户确认；用户可随时补充或纠正，系统应更新「已确认约束」并避免前后矛盾。
- **用户主动提问**：用户可用开放问句发起对话（例如意图、板块、风格类问题）；助手在文字答复之外，应给出与当前理解一致的**若干可选下一步**（澄清维度、候选方向、是否进入筛股等），供用户一键选择，减少从零打字成本。
- 快捷选项与自由输入并存：简单维度优先选项以减少输入成本；开放域描述允许短文本，系统负责抽取或追问确认；用户点选某选项后，须写入「已确认约束」并驱动后续对话或生成候选。
- 可选展示轻量进度（例如「已确认维度 5/8」），非强制固定题序；支持「跳过当前追问」时采用下文**已定默认值**并在对话中明确说明。
- 对话收敛后输出：候选股票列表（默认 3～10 只，可配置）、每只的入选理由摘要、与**已确认约束**的对应关系；用户选定一只股票后进入单票择时分析。

#### 对话话题维度（产品已定）

以下为选股对话须收敛到的**固定维度**：结构化筛股维度 **D1～D8**（共 8 项），以及**不参与自动筛股**的补充文本槽 **D9**。助手追问、复述、快捷选项与「已确认约束」均只围绕下列维度展开；**不得依赖未在本文定义的隐含维度**作为筛股依据。用户自由描述须映射到 D1～D8 的枚举或落入 D9；不得静默扩展新的结构化筛股字段。

##### D1 交易市场（单选，必选，不可跳过无默认）

用户必须明确其一：`A 股` | `港股` | `美股`。未完成前不产出候选列表。

##### D2 行业范围（单选模式 + 多选行业）

- 模式二选一：`不限制` | `指定行业`。  
- 当模式为 `指定行业` 时，用户须从下列 **GICS 一级行业（中文）** 中选至少一类（可多选）：`通信服务`、`非必需消费`、`必需消费`、`能源`、`金融`、`医疗保健`、`工业`、`信息技术`、`材料`、`房地产`、`公用事业`。  
- 当模式为 `不限制` 时，不得将行业作为筛选条件。

实现侧：`不限制` / `指定行业` 与快照字段 `sector_mode` 取值 `unrestricted` / `specified` 一一对应。

**跨市场与 D2（产品已定）**：当 **D1** 为 `A 股` 时，D2 的行业选项即上文 **GICS 一级（中文）** 枚举。当 **D1** 为 `港股` 或 `美股` 时，用户所选 GICS 类须在服务端映射为**该市场数据源支持的行业或板块分类**（映射表在实现中单列、可版本化）；若某类在该市场无可映射或映射歧义，**不得静默筛空**，须在对话中说明并建议用户改为「不限制」或改选其他类。

##### D3 主题叠加（多选，可与 D2 任意组合）

可选零项或多项：`红利/高股息`、`新能源产业链`、`人工智能与硬科技`。无任何勾选时等价于「无主题叠加条件」。

##### D4 分析持有周期（单选，必选）

`日内至数日（仅作观点参考）` | `1～4 周` | `1～3 个月` | `3 个月以上`。

##### D5 风格偏好（单选，必选）

`价值` | `成长` | `动量/趋势` | `无明确风格偏好`。

##### D6 风险承受档位（单选，必选）

与择时模块一致：`保守` | `平衡` | `进取`。

##### D7 市值与流动性（单选，必选）

`不限制` | `偏大中盘与流动性` | `接受小盘高波动`。

##### D8 排除规则（多选，允许为空集合）

在数据源可计算前提下，可选：`排除 ST 与 *ST`、`排除流动性过差标的（低于产品设定成交额阈值）`、`排除高负债或杠杆显著异常标的`。空集合表示不额外启用上述排除项（法定或交易所层面强制排除仍以数据源与合规为准，单独在报告中披露，不写入本多选语义）。

##### D9 其他补充（可选，短文本）

仅用于记录无法用 D1～D8 枚举覆盖的说明性文字（例如个人关注名单、临时事件描述），**不参与自动筛股逻辑**；若其中包含可映射到 D1～D8 的语义，系统须追问确认并写入对应枚举，不得静默当作筛选条件。

#### 未确认或用户跳过时的默认值（产品已定）

仅适用于用户明确选择「跳过」或经多轮追问仍无法解析该维度的情形；**D1 不适用**。

- **D2**：模式视为 `不限制`（不选任何 GICS 行业）。  
- **D3**：视为无主题叠加（等价于零项勾选）。  
- **D4**：视为 `1～3 个月`。  
- **D5**：视为 `无明确风格偏好`。  
- **D6**：视为 `保守`。  
- **D7**：视为 `偏大中盘与流动性`。  
- **D8**：视为空集合（不启用可选排除项）。  
- **D9**：空。

#### 结构化偏好快照字段键（与 D1～D9 对应）

后端或客户端持久化的偏好快照须至少包含下列键名，取值与上文枚举一致；类型约定：`market`/`sector_mode`/`holding_horizon`/`style`/`risk_tier`/`cap_liquidity` 为字符串枚举，`sectors`/`themes`/`exclusions` 为字符串数组，`other_notes` 为可空字符串。

| 键名 | 对应维度 |
| ------ | ---------- |
| `market` | D1 |
| `sector_mode` | D2 模式：`unrestricted` \| `specified` |
| `sectors` | D2 行业多选；`sector_mode` 为 `unrestricted` 时为空数组 |
| `themes` | D3 |
| `holding_horizon` | D4 |
| `style` | D5 |
| `risk_tier` | D6 |
| `cap_liquidity` | D7 |
| `exclusions` | D8 |
| `other_notes` | D9 |

#### 输出要求

- 每条候选须附带**可复述的筛选依据**（对应对话中已确认的哪些约束项）。
- 若无法匹配足够标的，须在对话中说明缺口原因并建议放宽或修改哪一项约束，而非静默返回空列表。

#### 与择时模块的衔接

- 从选股结果进入择时时，须携带对话摘要（或结构化偏好快照），供择时侧风险闸门与评分参考，保证体验连贯。

### 5.1 输入层

- 股票标识：市场 + 代码（示例：US.AAPL、CN.600519）。
- 分析周期：日线主导，兼容周线确认。
- 用户偏好：风险级别（保守/平衡/进取）、预期持有周期。
- 可选上下文：事件窗口（财报日期、政策节点）。

### 5.2 信号层（择时核心）

系统基于技术面为主、基本面与事件面为辅，构建多维择时评分。

与 **第 6 章** 口径对齐：**第 6.1 节「结构与风险评分」**在实现上同时承载**价格结构、波动与风险**及可获得的**简明基本面/估值类特征**（如估值分位、财务稳健度摘要等，以数据源可稳定提供为限）；**「技术信号评分」**对应量价与趋势动量类；**「事件折扣评分」**对应财报与重大事件窗口。不在产品层单独拆「基本面」第三条分数柱，避免与 6.1 三维权重冲突。

- 趋势信号：中短期均线结构、趋势斜率、趋势一致性。
- 动量信号：RSI、MACD、价格动量持续性。
- 量价信号：放量突破、缩量回踩、背离识别。
- 结构信号：关键支撑阻力、箱体突破、前高前低关系。
- 波动信号：ATR、波动扩张收敛状态。
- 事件信号：财报或重大事件前后的风险折扣。

### 5.3 决策层

将多维信号转化为五态决策：

- **观望**：信号冲突或风险收益比不达标。
- **试仓**：趋势初步成立，适合小风险敞口跟踪验证。
- **加仓**：趋势确认且风险可控，可提升关注强度与风险敞口。
- **减仓**：上行动能衰减或风险上升，建议收缩风险敞口。
- **离场**：趋势破坏或风险位触发，建议停止跟踪该方向。

### 5.4 计划层（输出可执行化）

每次分析必须包含：

- 建议动作与原因摘要。
- 关注区间（价格带而非单点）。
- 风险位与观察目标位。
- 风险敞口建议（百分比区间）。
- 失效条件（价格、时间或事件触发）。
- 建议有效期（例如 T+3、T+5 交易日）。

### 5.5 复盘层

- 保存每次建议快照（输入、输出、时间戳、数据版本）。
- 到期后回填结果（命中、未命中、提前失效）。
- 统计策略层指标：命中率、平均盈亏比、最大回撤、建议稳定性。

**统计与命中口径（产品采纳）**：命中率、平均风险收益比（R 值）、失效触发率等（见 **10.2**）须在**同一数据版本**与**同一复权规则**下计算；复权方式（如前复权 / 后复权）及主行情源须在实现配置中单列并**版本化**，变更时不得静默混用历史序列。**命中**指在建议**有效期**（见 **5.4**）内，标的实际价格路径是否达到与当次报告字段可追溯对应的**方向性结果**（具体阈值或判定规则与 `analysis_id` 绑定、可回放）。**R 值**为窗口内有利潜在变动与不利潜在变动之比的实现定义，须与当次 **5.4** 所载风险位与观察目标位同源回溯。**提前失效**以 **5.4** 失效条件（价格、时间、事件）**任一**先满足为准。

### 5.6 股票分析 Agent 编排逻辑

本节描述**产品级**编排：将「股票分析」拆为两条由 **编排器（Orchestrator）** 驱动的链路——**选股对话链**与**单票择时分析链**。自然语言与推理可由大模型 Agent 承担，**枚举、权重、闸门与数据源契约**以本文与实现契约为准，优先于自由生成，保证可复现、可审计与合规。与前端消息形态、载荷字段的对齐见 **第 7 章**：选股对话 **7.6**，登录注册 **7.9**，订阅与会员 **7.10**，Landing 与法律静态页 **7.11**。

#### 5.6.1 编排器共性

- **公开路由（产品已定）**：**FR-015～FR-017** 所对应页面为**静态或可静态化**的公开资源，**不得**被鉴权中间件默认拦截；其 HTML 与元数据可由 CDN 或边缘缓存加速（策略见 **NFR-013**）。
- **滥用防护（NFR-014）**：对**游客**及未鉴权高频路径（尤其 **5.6.2** 编排入口、游客可访问的 **5.6.3**），须在网关或边缘层实施配额与限流，**早于**重量级子 Agent 展开，并与 **FR-014** 配置表可对照；避免仅靠业务尾端拦截导致成本失控。
- **网关前置顺序（产品已定）**：对需保护的资源，须先完成 **身份鉴权（FR-009～FR-011）**，再执行 **订阅与用量校验（FR-014）**，最后进入 **选股链（5.6.2）** 或 **择时链（5.6.3）**；游客仅允许命中「免费档最低权益」所声明的接口集合（见 **7.10**）。
- **会话级状态机**：维护当前阶段、在途子任务、已提交的结构化中间结果；对外与选股会话 API（7.6「前后端契约」）及单票分析 API 的入参/出参模型一致。
- **路由**：依据输入类型（用户自然语言、`suggested_actions` 的 `action_id`、直达分析请求）选择下一跳子 Agent 及**串行 / 并行**关系。
- **汇合（Merge）**：并行子任务完成后由编排器合并；冲突优先级约定为：**合规与硬规则 > 数据事实 > 模型生成内容**。
- **持久化与可观测**：关键中间态须可落库或会话存储，至少包含 `preference_snapshot`（选股链）、行情窗口标识、信号向量或摘要、`score_breakdown`、闸门结果码；与 **NFR-007、NFR-008** 一致。

#### 5.6.2 选股对话链（对应 FR-007、FR-008）

- **阶段（示例枚举，实现可映射为内部状态名）**：`intent_classify`（咨询 / 筛股）→ `slot_and_clarify`（收敛 **D1～D9**）→ `screen_candidates`（检索与排序）→ `handoff_ready`（用户选定标的，组装进入择时链的 payload）。
- **典型子 Agent 职责**（名称可映射实现，边界不可缺）：
  - **意图与分流**：判断咨询 vs 筛股；筛股路径才挂载检索与硬闸门相关子任务（与第 7 章分流一致）。
  - **对话理解与槽位**：将文本与点选事件映射为合法 `preference_snapshot`；非法值拒收并触发澄清。
  - **追问与选项生成**：缺槽或低置信时生成追问文案及 `suggested_actions`（与已定枚举对齐）。
  - **筛股执行**：在「可筛」闸门满足后访问只读数据与规则，产出 `candidate_stocks` 及与 **D1～D8** 可复述的对应关系。
  - **合规与话术**：风险提示、敏感组合审查；可与话术生成并行，但汇合时不得被覆盖。
- **硬闸门**：**D1 未完成**不得进入 `screen_candidates`；不得用 **D9** 自由文本替代未确认的枚举槽参与自动筛股。

#### 5.6.3 单票择时分析链（对应 FR-001～FR-004）

- **触发**：A）用户直接提交「市场 + 代码 + 分析参数」；B）自选股链 `handoff_ready` 携带 `preference_snapshot` / 会话引用。
- **阶段（示例枚举）**：`validate_input` → `fetch_market_context`（只读行情与上下文）→ `technical_compute`（与 **5.2 信号层** 对应）→ `score_aggregate`（与 **第 6 章** 权重一致）→ `risk_gate`（与 **6.2** 规则一致）→ `plan_generate`（与 **5.3、5.4** 五态及计划字段一致）→ `persist_and_respond`（结构化响应；内嵌触发 **FR-005** 存档；**FR-006** 所需块在此阶段或紧随的异步任务中组装）。
- **典型子 Agent 职责**：
  - **输入校验与标的解析**：代码、市场、周期、数据可得性；失败则短路返回可读错误，不进入重计算。
  - **数据访问**：封装数据源与时间窗口，输出规范化特征表或指标输入，避免向推理 Agent 注入无关大体量原文。
  - **技术信号**：计算趋势、动量、量价、结构、波动等；实现上可拆多 Worker，编排器仍归为阶段 `technical_compute` 的并行子任务并在汇合后进入评分。
  - **评分与分解**：产出总分及维度分解 `score_breakdown`，供闸门与解释卡片使用。
  - **风险闸门**：只读规则与 `score_breakdown`、事件标记、用户风险档；输出是否降级及原因码，驱动 **5.3** 可见结论。
  - **研究计划生成**：在闸门后生成结论、关注区间、风险位、敞口、失效条件与有效期；**结构化字段与叙述文本须同源**，避免两套 truth。
  - **复盘写入**：在 `persist_and_respond` 内或紧后触发存档与数据版本写入（**FR-005**）。
- **顺序硬约束**：`technical_compute` → `score_aggregate` → `risk_gate` → `plan_generate` 不得跳过或颠倒；`validate_input` 为任意路径首步。

#### 5.6.4 跨链衔接与上下文传递

- **Handoff 最小 payload**：`symbol`、`market`、`preference_snapshot`（完整或持久化引用）、可选 `dialogue_session_id`、`holding_horizon` 等已从快照继承的字段；已登录时须携带 **`user_id`（或等价主体标识）** 以便 **FR-005**、**FR-014** 与审计一致。
- **游客态跨链（产品采纳）**：未登录游客从选股链进入择时链时，**不**写入 **FR-005** 个人存档（与 **7.10** 一致）；`handoff` 可为一次性分析上下文；若用户随后登录，**不**自动将此前游客会话并入账号，除非产品另行提供显式「保存到账号」且用户确认（若做，须在隐私与条文中说明）。
- **择时链消费偏好**：用于预填 **5.1** 输入层（如风险档、周期），或触发与选股侧一致的保守默认；**禁止**用未结构化到 D1～D8 的对话原文直接参与计分或闸门。

#### 5.6.5 降级与失败

- 模型或某子 Agent 不可用：选股链可降级为**规则化追问树**（基于 D1～D9 与题库）；择时链在 `technical_compute` 若无可替代规则源则返回**明确不可用**与重试建议，禁止空响应。
- 任一步产出违反契约或枚举：编排器丢弃该步非合规生成、写审计日志，并走澄清或错误分支。

#### 5.6.7 导出与异步任务（FR-006）

- **同源原则**：`plan_generate` 产出的结构化块为 **Markdown / PDF** 的唯一事实来源；同步渲染下载或投递异步导出队列时，**不得**由独立二次推理生成与报告页冲突的文本。
- **异步导出**：若导出耗时超出交互阈值，编排器可将任务置为后台并返回任务标识；前端轮询或推送完成后提供下载链接，失败须可重试且文案与报告页错误态一致（与 **7.4** 一致）。
- **幂等**：同一 `analysis_id`（或等价主键）重复请求导出应覆盖或复用同一逻辑版本，避免用户持有相互矛盾的两份「同一时刻分析」。

#### 5.6.8 编排健康度与运行时信号

- 编排器或网关应能输出**运行时语义**（供 NFR-003 与运维使用）：例如模型路由是否可用、数据延迟是否超阈、是否已启用选股/择时降级分支；**不要求**在本文定义具体 HTTP 路径或字段名，但实现须与可观测性方案一并落地。
- **超时与熔断**：子 Agent 调用须有单独超时与重试上限；熔断时落日志并走 **5.6.5** 降级路径，不向用户返回未处理异常栈。

## 6. 择时评分框架（产品口径）

### 6.1 评分构成

- 技术信号评分：60%
- 结构与风险评分：25%
- 事件折扣评分：15%

与 **5.2** 对齐说明：**「结构与风险评分」**合并体现价格结构、波动风险及数据源可支持的**简明基本面/估值摘要**；不在本层再单列「基本面%」，避免与 5.2 文字「基本面为辅」割裂。

总分范围 0-100，映射到决策建议：

- 0-39：离场/严格观望
- 40-54：观望
- 55-69：试仓
- 70-84：加仓
- 85-100：强势持有或分批加仓（仍受风控上限约束）

**与五态（5.3）及闸门（6.2）对齐（产品采纳）**：接口与 UI 的**五态枚举**仍为 **5.3** 所列五种，**不**因分数段再增第六种。0～39 分数段在文案上可称「偏严格观望」，映射到五态时须二选一：**若已触发 6.2 所列硬降级条件**（如关键支撑下破且未有效收复），输出 **离场**；**否则**输出 **观望**。85～100 分数段在 UI 上仍归入 **加仓**（与「强势持有或分批加仓」表述一致），并受 **6.2** 闸门与 **5.4** 风险敞口上限约束。

### 6.2 风险闸门规则

即使总分较高，满足以下任一条件时降级建议：

- 单笔风险超出用户风险档允许范围。
- 财报前高不确定窗口且历史波动显著放大。
- 关键支撑下破且未出现有效收复。

### 7.6 问答对话式选股交互（FR-007、FR-008）

以下对应 **FR-007、FR-008**；D1～D9 与快照键见第 5 章。**阶段划分、子 Agent 职责、跨链 handoff 与降级**等产品级编排逻辑见 **5.6**；本节侧重**用户可见交互**与**前后端载荷契约**。

本节约定：**前端**为 AI 对话式交互；**后端**通过 **Agent 编排**（多智能体协同与状态机）驱动理解、澄清、筛股与出话，而非单一静态脚本树。

#### 前端：AI 对话形态

- 以**会话消息流**为主界面：用户消息与助手消息按时间线展示，支持自然语言输入与点选混合交互。
- 助手单条消息由「可读文本」与「结构化附加载荷」组成：载荷中携带本条的**建议选项**（选项 ID、展示文案、互斥/多选语义等），由前端渲染为 chips、按钮或列表；用户点选后作为显式事件回传后端，与纯文本输入等价参与会话状态更新。
- 可选支持流式输出助手文本（字级或句级），但**同一条消息内选项列表须在载荷完整后再渲染**，避免用户点到半成品选项。
- 单条助手消息内信息密度适中，避免一次抛出过长表单；复杂说明可折叠或拆成多条连续助手消息。
- 快捷选项文案简短；术语可在助手消息内简短释义或链到帮助说明。
- 用户以问句或模糊描述发起时，助手除文字解释外，应附带**可选操作或澄清项**（视场景一般不少于 2 项），选项须与上文语义一致，互斥或并列关系在文案或载荷字段上可区分。
- 提供「重新开始对话」与「仅修改某项已确认偏好」两种修正路径；修改后须重新走编排流程并重新生成或显式标注候选列表已更新。
- **会话顶栏「已确认偏好」摘要**：与 `preference_snapshot` 一致，以 D1～D8 的简短标签展示（D9 若有则折叠展示），可点击展开详情；用户始终可见「系统当前如何理解我」，降低失忆感与误操作。
- **助手消息带来源一句**：在追问或给选项前，用一句话复述依据（例如「根据你已选：A 股、信息技术…」），与可解释性要求一致。
- **咨询与筛股分流**：首屏或首轮提供弱分流入口（如「随便问问」与「帮我选股」），或由编排器对用户首条消息做意图分类；**纯咨询**可多轮解释、少推筛股类动作；**要筛股**则尽快收敛快照并触发检索，避免无效拉长会话。
- **断线续聊**：会话恢复时展示「上次进度」摘要，并提供「继续」与「重新开始」；继续须载入编排持久化状态，与 NFR-007 一致。
- **流式输出时的选项区**：文本可流式展示；**选项列表与快照变更类 UI**在 `suggested_actions` 载荷未完整前显示明确加载态（如「正在整理可选项…」），禁止半成品可点状态。

#### 后端：Agent 编排（必选）

- **统一入口**：对话会话 API 接收 `session_id`、用户文本与/或 UI 事件（如某选项 ID）、可选上文摘要指针；返回本轮助手消息（文本 + 选项载荷）、更新后的 `preference_snapshot`（与 D1～D9 一致）、以及是否已满足「可出候选」闸门。
- **编排器（Orchestrator）**：维护会话级状态机（澄清中 / 待确认快照 / 已出候选等），按规则与模型输出**决定下一跳调用哪个子 Agent、是否并行、以何顺序合并结果**，并将合并结果编排为对用户可见的一条或多条助手消息；须支持**咨询 / 筛股**意图分类后的分支策略（与前端弱分流入口一致），避免纯咨询会话被过早拖入筛股动作链。
- **建议子 Agent 职责划分**（产品级边界，实现可选用任意框架，但能力须覆盖）：
  - **对话理解与槽位填充**：将用户自然语言与点选事件映射、校验并写入 D1～D9 合法取值；发现冲突时生成澄清话术与选项。
  - **追问与选项生成**：在必填槽位缺失或置信不足时，生成追问与**可点选**的候选答案（与已定维度枚举对齐）。
  - **标的检索与排序**：在 D1 已确认且快照满足最小可筛条件时，访问行情与基本面数据源（或检索服务），产出候选列表及与快照字段的对应说明。
  - **合规与话术**：统一拼接风险提示、禁止收益承诺表述；对敏感表述做拦截或改写建议。
- **状态持久化**：编排层须持久化消息序列摘要、`preference_snapshot`、待澄清维度列表及编排决策日志（至少会话级），以满足 NFR-007 与 AC-009。
- **降级策略**：模型或子 Agent 不可用时，须降级为**规则化追问**（基于已定维度与题库）或返回明确错误说明，**禁止**返回空消息或静默失败。

#### 状态可见、结果呈现与合规（产品采纳）

- **动作类选项（高阶）**：除澄清类选项外，`suggested_actions` 应支持显式「下一步动作」类 `action_id`（由产品词表定义，如：按当前条件生成候选、先收紧风险再筛、仅重开某一维度修正等），编排器识别后走对应子流程，减少用户在细枝末节上反复绕行。
- **一键补全未填维**：当信息已部分足够但未满足 8 维必填时，可提供「用已定默认值补全剩余项并继续」类动作；须在助手文案中列出将采用的默认项，且用户可在下一步用「仅修改某项偏好」撤销或覆盖。
- **候选阶段结果区**：`conversation_phase` 为 `candidates_shown` 时，前端以**独立结果卡区域**呈现（可与消息流同页但版式分区），包含列表、与快照关键字段的对照、「为什么入选」一行理由（须可映射回 D1～D8）；底部保留一句式对话入口（如换一批、更偏某主题），事件仍走同一会话 API。
- **维度级修正（外科手术式）**：「仅修改某项已确认偏好」须尽量只使**从该维度起**的编排分支重算，其前已确认且未冲突的维度默认保持不变，除非编排器检测到依赖冲突须一并追问。
- **合规前置与敏感二次提示**：进入选股对话时展示可折叠的短风险提示；当选项或快照涉及「进取」「小盘高波动」等敏感组合时，助手须追加**一句轻量提示**（非长篇说教），仍禁止收益承诺。

#### 前后端契约（摘要）

- 每轮响应须机器可读地携带：`assistant_text`、`suggested_actions[]`（可空）、`preference_snapshot`（部分或全量）、`conversation_phase`（如 `clarifying` | `ready_to_screen` | `candidates_shown`）、可选 `candidate_stocks[]`（进入候选阶段后）。
- `suggested_actions` 中每一项须含稳定 `action_id` 与展示文案，便于埋点、回放与 AC-009 验收；**高阶动作**与澄清类选项共用同一数组时，须用类型字段或 `action_id` 前缀区分，便于前端分区渲染（如主操作区 vs 消息内 chips）。

#### 无障碍与多端

- **键盘与读屏**：`suggested_actions` 的每一项须提供可聚焦的控件与屏幕阅读器可读文案（与 `action_id` 绑定），焦点顺序与消息流阅读顺序合理。
- **移动端**：关键选项建议采用**底部固定操作区**承载主要 `suggested_actions`，避免跟在长文本后需长距离滚动才可点选。

### 7.7 文案与表达规范（全站）

- 结论必须使用动作动词：观望、试仓、加仓、减仓、离场。
- 禁止使用收益承诺型语言。
- 对不确定性给出明确表达，如「等待突破确认后再执行」。
- **服务定性（产品采纳）**：对外文案与报告脚注中，将本产品表述为**研究信息与分析工具**，**非**证券投资咨询、非资产管理、非代客决策；不得声称替代持牌证券服务机构意见（具体措辞由合规审定，与 **9.1** 一致）。

### 7.8 进阶能力交互占位（FR-101～FR-103，MVP 后）

- 分时与日线联合、策略参数模板、事件前后折扣等上线后，须在 **7.3** 报告与 **7.4** 导出中增加对应模块或字段，并在后续文档版本补充交互细则；**未上线前**不在主导航以完整功能形态占位，避免误导。

### 7.9 登录、注册与账号（FR-009～FR-011）

- **统一账号模型**：同一用户可通过多种方式登录，**手机号**作为账号主键之一；微信扫码成功后与已存在或未注册账号的**绑定 / 自动建号**规则在实现设计中写死，避免重复身份。
- **入口与形态**：提供独立「登录 / 注册」页或模态；三种方式以 **Tab、分步或并列卡片** 呈现，切换时保留已输入手机号（脱敏显示）等合理状态，减少重复输入。
- **手机号 + 密码（FR-009）**：注册须校验手机号格式与密码强度（最低规则如长度与复杂度由安全需求定义）；登录失败须模糊提示（防枚举）；提供「忘记密码」并通过 **FR-010** 短信验证码完成身份校验后重设。
- **手机号 + 验证码（FR-010）**：登录与注册关键步骤须支持短信验证码；展示倒计时与重发限制；错误次数过多须冷却或人机校验（产品层要求，具体策略见 NFR）。
- **微信扫码（FR-011）**：PC 端展示可刷新二维码与过期提示；移动端可用跳转授权；授权失败与取消须有明确文案与重试入口。
- **登录后**：展示账号标识（手机号脱敏、微信昵称可选）；提供「退出登录」；与 **FR-005** 等个人数据读取须带用户上下文且仅本人可见。
- **隐私与注销**：提供指向 **FR-016** 独立「隐私政策」页面的链接（与仅内嵌摘要二选一或并存时，须保证**全文**可在 **7.11** 访问）；提供「账号注销」入口与后果说明（与 **FR-009**、**NFR-012** 一致）。

## 8. 功能需求（FR）

### 8.1 核心功能

股票分析相关的 **Agent 编排**（选股链、择时链、衔接、导出异步与健康信号）产品定义见 **5.6**（含 5.6.7、5.6.8）。

- **FR-001**：支持单只股票分析请求。
- **FR-002**：返回五态择时决策与结构化评分。
- **FR-003**：返回研究计划字段（关注区间、风险位、观察目标位、风险敞口、失效条件）。
- **FR-004**：返回置信度和风险等级。
- **FR-005**：支持建议存档和后续复盘查询。
- **FR-006**：支持单次分析报告下载，格式至少包含 `Markdown` 与 `PDF`。
- **FR-007**：提供基于 **AI 对话**的选股能力：前端会话消息流与结构化选项载荷；后端 **Agent 编排**（编排器 + 理解/追问/检索/合规等子 Agent）驱动多轮对话；支持用户主动提问与系统主动追问；助手在理解用户意图后**可给出可选答案项**（快捷选项）供点选，并与文本输入并存；将对话收敛为 D1～D9 快照后输出候选列表及依据，并支持选定标的后发起单票择时分析。
- **FR-008**：选股对话 UI 须实现 **7.6** 各小节中的产品采纳项：已确认偏好摘要区、咨询/筛股分流、候选独立结果区与入选理由映射、断线续聊入口、流式时选项区加载态、高阶动作与一键补全未填维、维度级修正入口、合规前置与敏感轻提示；键盘与读屏可达、移动端底部主操作区。
- **FR-009**：支持使用 **中国大陆手机号（11 位）+ 密码** 完成注册与登录；支持修改密码与通过短信验证码验证身份后的**找回密码**流程；支持**账号注销**申请（注销后果、数据删除与保留范围见 **NFR-012** 与届时隐私政策）。
- **FR-010**：支持使用 **手机号 + 短信验证码** 完成登录；支持在注册或安全操作中发送与校验短信验证码（发送频率、有效期、单日上限须满足 **NFR-010**）。
- **FR-011**：支持 **微信扫码登录**（网页扫码或微信内授权等形态以实现设计为准）；须支持与 **FR-009 / FR-010** 所标识账号的绑定或首次登录自动建号，且同一手机号与同一微信 OpenID 不得对应多个冲突账号（合并规则在数据模型中单列）。
- **FR-012**：提供**订阅套餐**定义与展示：至少免费档与一档付费档；每档权益说明可对照核心功能；用户可查看当前订阅档位与有效周期。
- **FR-013**：支持用户通过**支付渠道**完成订阅**新订与续费**（首期实现至少支持微信支付；订单、回调、对账须满足 **NFR-011**）；可选支持自动续费开关，规则须在用户协议或说明页明示。
- **FR-014**：提供基于订阅档位的**权益与用量控制**：对 **FR-001～FR-008** 中需限量的能力在服务端强制校验；**游客**对 **FR-001** 与 **FR-007/008** 的限额须与 **NFR-014** 一致并在网关或业务层强制；**免费档最低权益**不得低于 **7.10** 所列（以免与 **AC-001** 等冲突）；到期降级、超限拦截与可升级提示一致；各档位限额以可配置策略落地，与 **AC-014、AC-015** 可对齐。
- **FR-015**：提供 **Landing 页**作为产品**默认站点首页**（根路径）；承载品牌、合规摘要、核心功能入口与基础 SEO（见 **7.11**）。
- **FR-016**：提供可公开访问的**隐私政策**静态页，内容与 **NFR-012** 及实际数据处理一致，可展示更新日期或版本号（见 **7.11**）。
- **FR-017**：提供可公开访问的**用户服务条款**静态页，内容与账号、订阅、支付及免责等一致，可展示更新日期或版本号；与注册及支付勾选联动（见 **7.11**）。

- **FR-102**：支持策略参数模板（保守/平衡/进取）。
- **FR-103**：支持事件前后自适应风险折扣。

## 9. 非功能需求（NFR）

- **NFR-001 性能**：单票分析主接口在**行情与缓存就绪、不含大模型冷启动首次推理**的条件下，返回结构化报告的目标为 **P95 &lt; 3 秒**；含 LLM 叙述生成或冷启动时允许更高上限，须在接口契约中单列**超时与降级**（与 **5.6.5** 一致）。登录、支付回调、导出异步任务不适用本 3 秒口径。**选股对话编排接口**（含多子 Agent 与模型）不适用上述 3 秒同一数值；须在契约中单列**单轮用户可见响应**的 P95 目标或「首块可渲染内容」延迟上限，以及总轮询 / 流式超时，避免团队误将选股链与单票链混用同一 SLA。
- **NFR-002 稳定性**：同一输入在同一数据版本下结果可复现。
- **NFR-003 可观测性**：记录请求耗时、评分分布、错误率、建议分布。
- **NFR-004 可解释性**：每条动作建议须附带信号依据；**常态**下不少于 3 条。若处于 **5.6.5** 规则降级或数据严重缺失路径，允许少于 3 条，但须在 `依据卡片` 显式标注「依据不足 / 降级模式」并列出已有关键事实，不得伪造依据条数。
- **NFR-005 合规性**：明确“仅供研究参考，不构成投资建议”。
- **NFR-006 导出一致性**：`Markdown` 与 `PDF` 导出内容字段一致，不得缺失核心风险信息。
- **NFR-007 对话状态**：选股对话会话可恢复或可导出摘要（至少在后端或本地其一实现），避免长对话中断后从零重复；须与编排层持久化字段一致。
- **NFR-008 编排可观测**：记录编排阶段迁移、子 Agent 调用结果摘要（非必须落原文模型全量）、错误与降级路径，支撑排障与质量分析。
- **NFR-009 对话无障碍与多端**：选股对话中由 `suggested_actions` 渲染的控件须满足键盘可操作与屏幕阅读器可读；移动端须避免关键动作仅出现在长文本之后不可 discover 的区域（推荐底部固定主操作区）。
- **NFR-010 账号与安全**：密码须以不可逆方式存储（委托实现侧算法与盐策略）；短信验证码须具备发送冷却、单日次数上限、图形或行为验证码等防刷能力（最低限度在实现设计中落实）；微信 OAuth 回调与令牌传输须走 HTTPS；会话令牌须支持过期与刷新、登出失效；须记录关键安全事件（登录失败、改密、绑定变更）供审计。
- **NFR-011 订阅与支付安全**：支付回调须**幂等**、可验签、防重放；订单与订阅状态变更须可审计；金额与币种展示准确；不在日志中记录完整支付敏感信息；对账与异常订单须有后台处理路径（产品要求具备，界面可简）。
- **NFR-012 隐私与个人信息**：须提供**隐私政策**入口（以 **FR-016** 独立页面为权威载体之一）；向微信、支付等第三方共享或委托处理的类型须在政策中列示；支持用户**账号注销**申请，注销后按策略删除或匿名化业务数据（法律法规要求保留的除外）；支持法律要求范围内的**查询与导出**指引（具体交互可简化为工单或邮箱，但须在政策中承诺路径）。
- **NFR-013 静态与法律页**：**FR-015～FR-017** 页面须使用**稳定 URL**、可缓存、默认**无需登录**；须具备基础 **SEO** 元信息与合理标题层级；`robots` / 是否对部分页 `noindex` 由发布策略在 checklist 中定义；法律正文须可随版本迭代并保留可追溯版本号或更新日期。
- **NFR-014 滥用防护与公平使用**：在 **FR-014** 之外，对**游客**与**未付费**主体实施可配置的**限流与配额**（含单票分析、选股编排会话、同设备 / 同 IP 短时间重复请求）；超限须返回可理解文案并允许合法用户通过登录或人机验证继续；须记录关键拦截事件供运营与风控审计；与 **NFR-010** 短信防刷互补，**不**在日志中落全量敏感个人信息。

### 9.1 产品边界（强约束）

- 提供 **Landing 首页**及**隐私政策**、**用户服务条款**公开页（**FR-015～FR-017**、**7.11**、**NFR-013**），与账号、订阅流程链接一致。
- 提供**用户注册与登录**（**FR-009～FR-011**）与**订阅及支付用于订阅**（**FR-012～FR-014**、**NFR-011**）；**不包含**证券开户、代客理财及收益分成类收费（见 3.2）。
- 个人信息处理须符合 **NFR-012** 及适用法律法规；不提供超出研究工具范畴的投顾签约服务；产品形态为**证券研究信息与技术工具**，**不构成**证券投资咨询业务或资产管理服务（对外表述见 **7.7**）。
- 不提供下单入口，不发起任何真实交易请求。
- 不对接券商账户，不读取持仓、资产或委托状态。
- 不包含委托管理、成交回报、撤单等交易执行相关模块。
- 所有输出均为研究结论与风险提示，不构成交易指令。

### 9.2 合规附录占位（法务定稿）

下列主题须在上线前由**法务 / 合规**以独立附件或 **FR-016 / FR-017** 子章节定稿，本文仅要求产品与实现**预留入口与版本号机制**：**未成年人与限制民事行为能力用户**的规则（是否年龄门槛、监护人同意等）；**自动化决策 / 算法推荐**（若适用法律法规要求）的说明与用户控制；**营销短信、应用推送、邮件**等触达的**单独同意**与退订路径；与 **NFR-012** 一致的个人信息处理全量清单更新流程。

## 10. 指标体系（北极星与护栏）

### 10.1 北极星指标

- **北极星（产品采纳）**：**「从有效触达到完成一次可复盘分析」的比率**，定义为：在选定统计周期内，**完成至少一次**达到 **AC-001** 口径的单票择时报告（含游客或登录）的**去重用户数**（或设备维度，以实现一致为准）占**同期活跃用户数**的比例；可再拆分子漏斗（Landing→单票、选股→单票、登录转化率）。该指标**仅依赖**产品内事件，**不依赖**券商成交或持仓数据。
- **辅助观测（与北极星并列跟踪）**：单票报告**完成率**、**FR-006** 导出触发率、选股会话**进入候选并完成**的比例、订阅页**到达至支付成功**转化率。
- **可选自报（非北极星硬依赖）**：若后续产品提供「是否已按建议操作」等轻量反馈，可作为采纳意愿的**探索性**指标，须在隐私政策中说明用途，**不得**替代上述可观测北极星。

### 10.2 质量指标

- 建议命中率（有效期内达到预期方向）。
- 平均风险收益比（R 值）。
- 建议稳定性（同向连续建议比例）。
- 失效条件触发率（风险控制有效性）。

以上四项的**计算口径**须与 **5.5**「统计与命中口径」一致，避免报表与复盘页数值不一致。

### 10.3 护栏指标

- 单次最大建议回撤超阈值占比。
- 高置信度建议失败占比。
- 事件窗口误判率。

## 13. 风险与应对

- **数据质量风险**：行情缺失或延迟导致建议偏差。  
  应对：数据新鲜度检查与降级输出。

- **过度拟合风险**：历史表现好但实盘失效。  
  应对：滚动窗口回测与参数上限约束。

- **黑天鹅风险**：事件冲击使技术信号短时失真。  
  应对：事件窗口自动降杠杆或观望。

- **用户误用风险**：将研究工具视为保证收益工具。  
  应对：显著风险提示与默认保守档策略。

- **订阅与支付争议风险**：重复扣款、回调延迟导致状态不一致。  
  应对：**NFR-011** 幂等与对账；用户侧展示订单与订阅状态；客服或自助补单路径（实现阶段定义）。
